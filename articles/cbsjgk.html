Adobe Premiere Pro 硬件要求

CPU

处理器（或 CPU）是 Premiere Pro 工作站最重要的部件之一。 虽然 GPU 加速越来越受欢迎，但 CPU 通常会对整体系统性能产生更大的影响。 但是，请注意，理论上更强大的 CPU 并不总是更好，因为 Premiere Pro 可以有效利用的内核数量是有限的。 这就是为什么昂贵的双 Xeon 工作站不再是 Premiere Pro 好的选择的原因之一。 凭借现代 CPU 中可用的核心数量，相对标准的工作站 CPU 将以低得多的成本轻松胜过双 Xeon。

哪种 CPU 最适合 Premiere Pro？

适合 Premiere Pro 的最佳 CPU 很大程度上取决于用途和预算。 对于主要使用 H.264 和 H.265 (HEVC) 的工作负载，英特尔的 Core i9 12900K 做得很好——这要归功于其内置的 QuickSync 功能——而且价格非常合理。 另一方面，在处理 RED 素材时，AMD 的 Threadripper Pro 3975WX 和 3995WX 位居榜首。

CPU 核心越多 Premiere Pro 会越快吗？

在一定程度上，核心越多性能应该越好，尽管导出视频比实时播放更是如此。 但是，Premiere Pro 不能完美地利用更多的核心数量，因此双 Xeon 工作站等极高核心数量的系统往往比单 CPU 工作站的性能更差，因为它们单个核心的性能较低。

因此，我们强烈建议你直接查看不同 CPU 的基准性能，而不是尝试仅根据核心数（或任何其他规格）来选择 CPU。

Intel 还是 AMD CPU 更适合 Premiere Pro？

目前，在大多数情况下，AMD 在 Premiere Pro 中的实时播放和导出性能方面更强。 然而，英特尔的酷睿处理器（尤其是新的第 12 代型号）是不错的选择，因为它们的性能接近价格相似的 AMD 锐龙 CPU，但包括可用于处理 H.264 和 HEVC 媒体的快速同步，而不是依赖于 GPU 来完成这些任务。 这释放了显卡的性能。

显卡（GPU）

Adobe 在过去几年中越来越多地使用 GPU，而 Premiere Pro 尤其受益于现代 GPU。如果剪辑中包含依赖于 GPU 加速的效果，拥有显卡变得越来越重要。

此外，自 Premiere Pro 14.2 版以来，Adobe 在编码（导出） H.264/H.265 时增加了对基于 GPU 的硬件加速的支持，这可以将导出时间减少多达 5 倍！ 从 Premiere Pro 14.5 开始，H.264/HEVC 解码（主要用于实时播放）现在也能够利用 GPU。 这可以在使用这些编解码器时提供明显更流畅的编辑体验。

哪种 GPU（显卡）最适合 Premiere Pro？

哪种显卡最好取决于使用了多少 GPU 加速效果以及是否经常导出 H.264/HEVC。 对于大多数用户来说，即使你回到上一代到 2000 系列卡，各种 NVIDIA GPU 之间也没有太大区别。 但是，如果使用的 GPU 加速效果越多，希望能够以越快的速度导出到 H.264/HEVC，使用更高端显卡的好处就越大。

Premiere Pro 需要多少 VRAM（视频内存）？

一个帧需要多少 GPU 内存才能在 GPU 上进行处理取决于位深度和编码，但是你可以使用一个公式来计算使用 GPU 加速所需的内存大小 .

所需的显卡内存大约等于：高 × 宽/13.107。

如果剪辑太大且超过了可用的 GPU 内存量，Premiere 将回退到纯软件渲染。 以下是一些常见图像尺寸所需的 GPU RAM 示例。

一个 4096 × 2160 像素的 4K 视频帧大约需要 675 MB。

佳能 T2i 的静止图像，5184 × 3456 像素，大约需要 1.37 GB，超过了某些笔记本电脑系统上可用的 1 GB。

一个 8K 视频帧，8192 × 4320 像素，大约需要 2.7 GB。

佳能 EOS 5DS 的静止图像，8688 × 5792 像素，需要大约 3.84 GB，因此你需要一个至少 4 GB 的显卡。

通常，我们会从一个静止图像过渡到另一个静止图像，这会使需求翻倍。 在组合多个大型静止图像或多个 8K 视频剪辑时，即使在功能强大的系统上，也很容易耗尽所有可用的 GPU RAM。
